JUNÇÃO ESPACIAL / SPATIAL JOIN

Este geoprocesso, é similar a um Join entre tabelas alfanuméricas, permite transferir os atributos de uma camada para outra com base em uma característica comum. A diferença entre o “Join Tradicional” e um Join Espacial é que as características comuns consideradas, não são que “um” ou “muitos” atributos das tabelas possuam o mesmo valor (ex: chaves primárias), mas sim as características espaciais.

O geoprocesso Junção Espacial implementado pela extensão de geoprocessamento de gvSIG permite seguir dois tipos de critérios espaciais para estabelecer a Junção Espacial:

  • “Vizinho mais próximo” (relação 1->1). Atribui a um elemento da camada origem os atributos do elemento mais próximo da camada destino/relacionada. Caso o elemento mais próximo intersecte (ou esteja contido no caso de polígonos) ao elemento original, havendo por tanto várias interseções, o algoritmo tomará o primeiro elemento analisado das possíveis interseções.


  • “Está contido / Contido em” (relação 1->M). Relaciona um elemento da camada origem com vários elementos da camada destino (mais especificamente, com aqueles que são intersectados). Neste caso a camada origem não herdará os atributos da camada relacionada, sendo que a operação será muito parecida a do geoprocesso Dissolver. Para os M elementos relacionados com um elemento da camada de origem, se dará ao usuário a possibilidade de escolher uma ou várias funções de agregação (média, mínimo, máximo, somatório) que se aplicarão sobre os atributos numéricos da camada destino/relacionada.
